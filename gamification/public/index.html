gitBadges

The best way to motivate and reward the effort!

gitBadges is a api integrated into git, built to recognize contributors to a particular project, assigning a badge when a contributor reaches an objective.

What is a Badge?

A badge is a symbol or indicator of an accomplishment, skill, quality or interest. Badges have been successfully used to set goals, motivate behaviors, represent achievements and communicate success in many contexts. A “digital badge” is an online record of achievements, tracking the recipient’s communities of interaction that issued the badge and the work completed to get it. Digital badges can support connected learning environments by motivating learning and signaling achievement both within particular communities as well as across communities and institutions.

The list of existing badges can be found by clicking "badges" in the main menu.

How does Badges work?

Each time a developer makes a local commit, it is recorded by the api. Information about the author and the time and date of the commit is collected. Wanted the user (create if necessary) and assigns the last commit done.

When a new accomplishment is reached, it will automatically reflected with a badge on the website.
You can see the list of users with their badges by clicking "Users" in the main menu.

What is needed to use it?

The only requirement to use the api is to have the post-commit script installed in the local repository (/localrepository/.git/hooks)
